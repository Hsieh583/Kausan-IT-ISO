## 1. 目的
確保帳號生命週期、權限核准與特權使用符合最小權限與職務分離要求，並可追溯。

## 2. 適用範圍
- 內部員工、約聘、實習生帳號
- 第三方與供應商的臨時/遠端存取
- 所有系統（AD、ERP、Email、VPN 等）

## 3. 角色與責任（RACI）

| 活動 | 申請人 | 部門主管 | IT/系統管理員 | 資安 | 稽核 |
|------|--------|----------|---------------|------|------|
| 帳號申請/變更/停用 | R | A | C | C | I |
| 特權帳號核准 | C | A | R | C | I |
| 權限定期審查 | I | A | R | C | I |
| 例外/臨時權限核准 | C | A | R | C | I |

## 4. 程序流程（摘要）
1) 申請：以《帳號申請單》提出，主管核准後送 IT。  
2) 建立/變更：IT 依核准單執行，紀錄工單編號。  
3) 通知：申請人確認權限，初始密碼強制改密。  
4) 停用：離職/異動 4 小時內停用並回收資源。  
5) 審查：至少每半年進行權限審查並留存紀錄。  
6) 特權使用：須工單授權、會話記錄與事後檢視。

## 5. 控制重點
- 最小權限原則，嚴禁共享帳號。
- 帳號與權限變更須有可追溯的工單或紀錄。
- 特權帳號使用需事前授權與事後審查（含錄影/日誌）。
- 離職、異動須在 4 小時內停用帳號。

## 6. 例外管理
- 例外須事前經主管與資安核准，指定時限並記錄原因。  
- 例外期滿自動收回權限。

## 7. 記錄與保存
- 帳號申請與核准紀錄（保存 ≥3 年）
- 帳號權限審查紀錄（保存 ≥3 年）
- 帳號停用紀錄（保存 ≥3 年）
- 特權帳號使用紀錄（保存 ≥1 年）

## Appendix A：帳號申請與建立步驟

### 申請流程
1. 申請人填寫《帳號申請單》（FRM-ACC-001）
2. 部門主管審核並簽核
3. 涉及特權或敏感系統時，需額外經資安審核
4. IT收到核准單後於2個工作日內建立帳號

### 帳號建立檢查清單
- [ ] 確認申請單完整且已核准
- [ ] 依最小權限原則設定權限
- [ ] 設定初始臨時密碼（24小時有效）
- [ ] 啟用多因子驗證（若適用）
- [ ] 登記於帳號管理系統/清冊
- [ ] 通知申請人與主管

## Appendix B：權限審查與簽核指引

### 審查週期
- 一般帳號：每半年審查一次
- 特權帳號：每季審查一次
- 外部/臨時帳號：每月審查一次

### 審查流程
1. IT產生《權限審查清單》
2. 各部門主管確認人員與權限的必要性
3. 標示需要保留、變更或停用的帳號
4. IT執行變更並更新清冊
5. 資安抽查審核結果

### 審查檢核項目
- [ ] 帳號使用者仍在職且職務未變
- [ ] 權限符合目前工作需求
- [ ] 無長期未使用帳號（>90天）
- [ ] 共享帳號已有合理說明與補強
- [ ] 特權帳號使用有審計記錄

---

## Appendix C：離職/異動停用作業

### 停用時限
- 正常離職：離職日當天
- 異常離職：4小時內
- 內部異動：依職務變更需求

### 停用檢查清單
- [ ] 收到人資/主管正式通知
- [ ] 停用AD帳號（含VPN）
- [ ] 停用Email與應用系統
- [ ] 撤銷特權與管理員權限
- [ ] 回收實體資產（門禁卡、筆電、USB）
- [ ] 轉移重要資料（信箱、檔案）
- [ ] 更新帳號清冊狀態為「已停用」
- [ ] 記錄停用日期與執行人

### 保留期限
- 帳號資料保留3個月後刪除
- 郵件與檔案依資料保存政策處理
- 審計日誌保留依記錄保存政策

## Appendix D：特權帳號申請、核准與會話留存

### 特權帳號定義
- Domain Admin、Enterprise Admin
- 資料庫 SA/DBA
- 系統 root、Administrator
- 應用系統管理員
- 雲平台管理員

### 申請與核准
1. 填寫《特權帳號申請單》
2. 說明業務必要性與使用期限
3. 部門主管 + 資安 雙重核准
4. IT建立並納入密碼庫管理

### 使用與會話記錄
- 使用前須透過密碼庫取得憑證
- 記錄取用時間、目的、操作人
- 啟用會話錄影（PAM/Bastion）
- 使用後歸還並記錄完成時間
- 定期審查使用日誌

### 審計要求
- 每月檢視特權帳號使用記錄
- 異常活動（非工作時間、頻繁失敗）需調查
- 未使用超過90天的特權帳號需重新評估
