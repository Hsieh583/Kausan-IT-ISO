# PRO-INC-001 資訊安全事件管理程序

| 適用範圍 | 所有資訊安全事件（含可疑事件、弱點、隱私事件） |

---

## 1. 目的
建立一致的事件通報、分級、調查、處置與復原流程，降低事件影響並提升可追溯性。

## 2. 角色與責任（RACI）

| 活動 | 通報人 | 事件處理人 | 資安 | 管理層 | 稽核 |
|------|-------|-----------|------|-------|------|
| 事件通報 | R | C | C | I | I |
| 事件分級與指派 | I | R | A | C | I |
| 取證與調查 | I | R | C | C | I |
| 處置與復原 | I | R | C | C | I |
| 對外通報（客戶/主管機關） | I | C | C | A | I |
| 事後檢討與改善 | I | R | C | A | I |

## 3. 程序流程（摘要）
1) 通報：任何人發現事件皆可透過事件報告表/票務系統立即通報。  
2) 分級：依影響（資料、可用性、法規）分為 P1/P2/P3，決定指揮鏈。  
3) 控制：隔離受影響系統，保留日誌與記錄以便取證。  
4) 調查：收集證據、分析原因、界定影響範圍。  
5) 復原：依變更/作業程序執行修復，恢復服務並驗證。  
6) 通知：必要時依合約/法規對客戶或主管機關通報。  
7) 事後檢討：完成 RCA、改進計畫與責任分工，更新防範措施。

## 4. 控制要求
- 所有事件必須有工單或事件編號，可追蹤狀態與時戳。  
- 取證資料需確保完整性（採集、存放、交接紀錄）。  
- 重大事件（P1）須於 24 小時內向管理層通報，必要時啟動 BCP。  
- 對外通報需經管理層與法務/隱私顧問核准。

## 5. 例外管理
- 暫緩修復或非標準處置需記錄原因並經資安與管理層核准，設定期限。  
- 例外到期須重新審核。

## 6. 記錄與保存
- 事件報告表與事件日誌（≥3 年）
- 事件調查與分析紀錄（≥3 年）
- 變更管理紀錄（若涉及修復，≥3 年）
- 對外通報與客戶通知紀錄（≥3 年）

## Appendix A：事件分級與通報路徑

### 事件等級定義

| 等級 | 影響範圍 | 回應時間 | 通報對象 |
|------|---------|---------|---------|
| P1-重大 | 資料外洩、服務全停、法規違反 | 立即 | 管理層+資安+相關部門 |
| P2-高 | 部分服務中斷、惡意程式感染 | 4小時內 | 資安+IT+相關部門 |
| P3-中/低 | 可疑活動、政策違規 | 24小時內 | 資安+IT |

### 通報路徑
1. 發現人員 → 直屬主管 / IT Help Desk / 資安信箱
2. P1事件：同步通報管理層與資安委員會
3. 所有事件須登記工單編號並追蹤

## Appendix B：取證與證據保存流程

### 證據類型與收集
- 系統日誌（Event Log、Syslog、應用日誌）
- 網路流量（Packet Capture、Firewall Log）
- 記憶體映像（必要時）
- 硬碟映像（涉及惡意程式或入侵時）
- 郵件與通訊記錄
- 實體證據（USB、紙本等）

### 保全步驟
1. 立即隔離受影響系統（斷網不斷電）
2. 記錄時間戳、操作人員、初始狀態
3. 使用寫保護裝置進行資料採集
4. 儲存於安全位置並加密
5. 建立監管鏈紀錄（Chain of Custody）

---

## Appendix C：惡意程式/入侵/拒絕服務處置手冊

### 惡意程式處置
1. 隔離：中斷網路連線
2. 掃描：使用更新的防毒工具全面掃描
3. 清除：移除惡意程式與殘留檔案
4. 修補：更新系統與應用程式補丁
5. 監控：持續監控是否再次感染

### 入侵處置
1. 確認入侵點與影響範圍
2. 變更所有相關帳號密碼
3. 檢查後門與持續性機制
4. 復原至乾淨狀態或重建系統
5. 強化存取控制與監控

### 拒絕服務攻擊
1. 啟動流量清洗服務（DDoS防護）
2. 限流與黑名單設定
3. 聯繫ISP或雲服務商協助
4. 評估是否需要暫時關閉服務

---

## Appendix D：對外通報與法規時限檢核

### 通報時機與對象
| 情況 | 通報對象 | 時限 | 法規依據 |
|------|---------|------|---------|
| 個資外洩 | 當事人、主管機關 | 72小時內 | 個資法 |
| 客戶資料事件 | 客戶、合約方 | 依合約約定 | 合約義務 |
| 重大系統中斷 | 相關利害關係人 | 即時 | 服務承諾 |

### 通報內容
- 事件性質與發生時間
- 影響範圍與人數
- 已採取的措施
- 聯絡窗口資訊
- 後續處置計畫

## Appendix E：事件報告表（FRM-INC-001）

**資訊安全事件報告表**

| 欄位 | 填寫說明 |
|------|---------|
| 事件編號 | 由系統自動產生或IT指定 |
| 通報日期時間 | 發現並通報的時間 |
| 通報人 | 姓名、部門、聯絡方式 |
| 事件發生時間 | 估計或確認的事件發生時間 |
| 事件類型 | □ 惡意程式 □ 入侵 □ 資料洩漏 □ 服務中斷 □ 其他 |
| 事件等級 | □ P1-重大 □ P2-高 □ P3-中/低 |
| 影響系統/資產 | 受影響的系統、應用或資料 |
| 事件描述 | 詳細說明事件經過 |
| 初步影響評估 | 資料、服務、法規、聲譽等影響 |
| 已採取措施 | 隔離、掃描、通知等 |
| 建議處置 | 初步建議的後續行動 |
| 主管簽核 | 部門主管確認 |

**填表說明**：
- 任何人發現可疑事件均可填寫
- 盡可能詳細描述，有助於調查
- P1事件請以電話+表單雙重通報
- 表單送交：資安部門或IT服務台
