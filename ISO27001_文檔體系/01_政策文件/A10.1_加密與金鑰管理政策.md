## 1. 目的
保護敏感資訊免受未授權存取，透過加密與金鑰管理確保資料機密性與完整性。

## 2. 加密應用範圍

### 2.1 強制加密
- **傳輸中**：
  - HTTPS/TLS 1.2+ (所有Web應用)
  - SSH (遠端登入)
  - VPN (遠端存取)
  - 郵件加密 (機密郵件)
  
- **儲存中**：
  - 資料庫機密欄位 (密碼、金鑰)
  - 備份媒介 (含離線備份)
  - 筆記型電腦與USB (BitLocker/FileVault)
  - 雲端敏感資料

### 2.2 建議加密
- 內部敏感檔案
- 客戶資訊
- 個人資料

## 3. 加密標準

| 用途 | 演算法 | 金鑰長度 | 說明 |
|------|-------|---------|------|
| 傳輸加密 | AES-256 / RSA-2048 | 256 / 2048 bits | NIST認可 |
| 儲存加密 | AES-256 | 256 bits | FIPS 140-2 |
| 雜湊 | SHA-256 / SHA-512 | N/A | 不可逆 |
| 已淘汰 | MD5, DES, SSL 3.0 | - | 禁止使用 |

## 4. 金鑰管理

### 4.1 金鑰生命週期
1. **生成**：在安全環境中產生，不可人工創建或共享
2. **存儲**：使用金鑰管理系統 (KMS) 或硬體安全模組 (HSM) 保護
3. **使用**：僅在授權應用與人員使用
4. **輪替**：定期更換 (建議年度或發現漏露時)
5. **停用**：廢棄金鑰妥善銷毀

### 4.2 金鑰保護
- 不得以明文存放於代碼、配置檔、備註
- 存儲於金鑰庫 (AWS Secrets Manager, Azure Key Vault, HashiCorp Vault)
- 備份加密且異地存放
- 存取需多因子認證與審計日誌

### 4.3 金鑰分離
- 加密金鑰 ≠ 驗簽金鑰
- 資料金鑰 ≠ 主金鑰
- 生成方 ≠ 使用方（可行時）

## 5. 特殊場景

### 5.1 公鑰基礎設施 (PKI)
- 使用認證單位 (CA) 發行的憑證
- 憑證有效期建議 ≤ 3年
- 定期監控憑證到期日
- 發現漏露須立即撤銷

### 5.2 雲端加密
- 優先使用雲平台原生加密 (AWS KMS, Azure Key Vault)
- 若使用客戶自帶金鑰 (BYOK)，需嚴格管理
- 多雲環境需統一金鑰管理策略

### 5.3 法規豁免
- 如因系統限制無法加密，需風險評估與補強
- 豁免需資訊長與資安核准
- 設定到期日並定期複審

## 6. 日誌與監控
- 記錄金鑰使用與存取
- 監控未加密的機密資料傳輸
- 追蹤金鑰輪替與撤銷
- 定期分析異常存取

## 7. 違規處置
- 發現密碼洩露：立即更換
- 金鑰漏露：撤銷並重新生成
- 傳輸未加密：立即糾正與通報

## 8. 審計與合規
- 每季檢查加密實施情況
- 驗證金鑰無被未授權取用
- 確保加密演算法未過時
- 符合法規要求（如個資法）
