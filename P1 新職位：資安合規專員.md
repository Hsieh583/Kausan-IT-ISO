# 新職位：資安合規專員

## 工作清單與平台對應

設計原則：將任務集中於 1～3 個主要平台，避免依賴個人記憶。

---

## 一、平台分工總覽

### 最終目標平台數

* 平台 A：Wazuh（資安／事件／合規紀錄核心）
* 平台 B：既有系統管理平台（NAS / Veeam / SQL / NVR）
* 平台 C：資產與異常追蹤台帳（Excel / SharePoint / 自建系統）

Wazuh 為每日必查主平台；其餘平台僅於需深入或追蹤時使用。

---

## 二、工作清單與平台對應表

### 1. 資安事件與系統異常（每日主線）

| 工作項目                | 是否高頻 | 建議主平台 | Wazuh 的角色 |
| ------------------- | ---- | ------- | -------- |
| AD 登入 / 帳號異動        | 每日   | Wazuh  | 主事件來源    |
| 核心 Server 登入 / 服務異動 | 每日   | Wazuh  | 主事件來源    |
| 防火牆 / VPN 異常        | 每日   | Wazuh  | 主事件來源    |
| 非上班時間操作             | 每日   | Wazuh  | 規則告警     |
| 異常事件備註 / 關聯         | 每日   | Wazuh  | 事件標記與查詢  |

上述項目需以 Wazuh 為主，以避免人力分散。

---

### 2. CCTV / NVR / NAS 監控

| 工作項目          | 是否高頻 | 建議主平台     | Wazuh 的角色   |
| ------------- | ---- | --------- | ----------- |
| CCTV/NVR 存活確認 | 每日   | Wazuh    | 存活與心跳紀錄   |
| 錄影異常（未錄/中斷）   | 事件型  | Wazuh → NVR | 發現來源       |
| NAS 存取異常      | 每日   | Wazuh    | 關鍵資料夾事件   |
| NAS 容量/服務異常   | 每週   | NAS 管理介面  | 事件紀錄       |

原則：不在 Wazuh 查看畫面；僅於 Wazuh 確認是否正常。

---

### 3. SQL / 核心系統

| 工作項目        | 是否高頻 | 建議主平台 | Wazuh 的角色 |
| ----------- | ---- | ------- | -------- |
| SQL 服務啟停    | 每日   | Wazuh  | 主事件      |
| 異常登入 / 權限異動 | 事件型  | Wazuh  | 主事件      |
| 資料異常需追蹤     | 事件型  | SQL 工具 | 留痕        |

Wazuh 僅記錄人員、時間與異常狀態，不分析 SQL 語句。

---

### 4. Veeam / 備份任務

| 工作項目    | 是否高頻 | 建議主平台 | Wazuh 的角色 |
| ------- | ---- | ------- | -------- |
| 備份成功/失敗 | 每日   | Wazuh  | 任務結果摘要   |
| 還原操作紀錄  | 事件型  | Wazuh  | 稽核留痕     |
| 備份設定異動  | 低頻   | Wazuh  | 變更紀錄     |

此類資訊為管理階層常詢問重點，需確保不遺漏。

---

### 5. 資產紀錄 / 合規台帳

| 工作項目   | 是否高頻 | 建議主平台 | Wazuh 的角色 |
| ------ | ---- | ------- | -------- |
| 設備清冊更新 | 每月   | 資產台帳  | 不負責      |
| 設備存活驗證 | 每日   | Wazuh  | 驗證來源     |
| 設備異常歷史 | 查詢型  | Wazuh  | 歷史證據     |

資產資料維持於台帳；存活與異常紀錄由 Wazuh 提供。

---

### 6. 異常事件追蹤

| 工作項目   | 是否高頻 | 建議主平台 | Wazuh 的角色 |
| ------ | ---- | ------- | -------- |
| 事件開立   | 事件型  | 追蹤台帳  | 事件來源     |
| 事件進度追蹤 | 持續   | 追蹤台帳  | 留存結果     |
| 結案證據   | 結案   | Wazuh  | 原始紀錄     |

Wazuh 作為證據庫，工單與追蹤功能留在台帳系統。

---

## 三、三大平台定位

### A. Wazuh（每日檢視）

* 發生事件總覽
* 異常狀態與告警
* 事件時間線

### B. 各系統原廠平台（按需使用）

* CCTV 畫面
* NAS 設定
* SQL 詳細內容
* Veeam 還原操作

### C. 異常與資產追蹤台帳（治理）

* Excel / SharePoint / 自建系統
* 開案、追蹤、結案紀錄

---

## 四、設計對人力請求的說明

此職位以 Wazuh 作為集中事件入口，其他平台僅於必要時深入，能兼顧效率、稽核要求與交接性。

---

## 五、職責定義

此角色負責確保事件不被遺漏，重點在監控與稽核，而非工程開發。
