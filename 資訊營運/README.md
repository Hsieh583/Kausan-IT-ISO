# 資訊營運 (IT Operations & Service Management)

**定位**: IT 服務管理與營運優先層級框架  
**對應標準**: ISO 9001:2015 (品質管理) + ISO 27001:2022 (資訊安全管理)  
**目標用戶**: IT 經理、資安主管、營運管理者

---

## 📋 資料夾目的

本資料夾存放 **IT 服務管理框架** 與 **營運優先層級** 的核心文件，用於：

1. **定義 IT 部門的服務範疇** — 不是技術清單，而是內部服務目錄
2. **建立品質管理體系** — 逐層提升 IT 成熟度（L1-L5）
3. **連結 ISO 9001 與 ISO 27001** — IT 的品質 + 安全雙軌管理
4. **指導採購與投資決策** — Make vs Buy 決策框架

**核心理念**: 不中斷、不爆炸、出事有訊號能處理、人用起來順、最後才是文件與證書

```
傳統模式：文件 → 合規 → 實施 → 營運
本目錄模式：營運穩定 → 防禦到位 → 體驗優化 → 自動化 → 文件驗證
```

---

## 🎯 核心原則

### 1. 營運優先，合規驗收
- ISO 27001 是「最終驗收標準」，不是「設計起點」
- 先把系統做穩定、做安全、做好用，文件自然產生

### 2. IT 是內部服務部門（非技術支援）
- IT 提供 7 項核心數位服務（對標 Azure / AWS / ITIL）
- 服務有明確的客戶、SLO、KPI
- Make vs Buy 決策框架：優先採購成熟服務

### 3. 分層建設，逐級提升（L1-L5）
- **L1**: 可靠性與容灾（DR/HA）- 基礎層
- **L2**: 防禦與監控（Defender/Logs）- 保護層
- **L3**: 使用者體驗（工具/流程）- 便利層
- **L4**: 工具與自動化（效率提升）- 效率層
- **L5**: 文件與合規證書（驗收層）- 驗證層

### 4. 成熟度驅動，持續改進
- 每個層級有清晰的成熟度指標（Level 1-5）
- 月度追蹤，確保穩定向上提升
- 避免「看起來合規，實際上脆弱」的假象

**檔案**: [IT Service Catalog_服務語言版.md](./IT%20Service%20Catalog_服務語言版.md)

**內容**:

**7 項核心服務**:
1. **Identity & Access Service** — 讓對的人在對的時間存取對的系統
2. **Compute & Application Service** — 穩定可用的系統執行環境
3. **Data Protection & Recovery Service** — 資料不會不見，出事也救得回來
4. **Security Monitoring & Incident Response Service** — 出事有訊號，能快速處理
5. **Network & Connectivity Service** — 人跟系統之間不被網路卡住
6. **Productivity & Automation Service** — 讓人少做重複事，工作跑得更快
7. **IT Governance & Visibility Service** — 讓管理層知道：IT 是被管控的

**適用對象**: IT 經理、服務負責人、業務部門

## � 使用指引

### 對新 IT 經理

**閱讀順序**:
1. 先讀 [A5 IT 服務管理政策](../ISO27001_文檔體系/01_政策文件/A5_IT服務管理政策.md) — 理解「為什麼」
2. 再讀 [IT Service Catalog](./IT%20Service%20Catalog_服務語言版.md) — 知道「提供什麼」
3. 最後讀 [L1-L5 框架](./L1-L5_營運優先層級框架.md) — 知道「怎麼做」

**使用方式**:
- 用 **IT Service Catalog** 規劃投資優先級
- 用 **L1-L5 框架** 評估當前成熟度
- 用 **月度進度追蹤** 監控改進進度

### 對高層管理者

**關鍵文件**: [A5 IT 服務管理政策](../ISO27001_文檔體系/01_政策文件/A5_IT服務管理政策.md)

**重點關注**:
- IT 提供哪 7 項服務？每項服務的客戶與價值是什麼？
- IT 的 KPI 是什麼？如何衡量 IT 的品質？
- Make vs Buy 決策框架是否合理？

### 對稽核人員

**關鍵對應**:
- **A5 IT 服務管理政策** → ISO 9001 第 8 章（營運）+ ISO 27001 A5（組織控制）
- **L1-L5 框架** → ISO 27001 各控制項（A12, A17, A13 等）
- **IT Service Catalog** → ISO 9001 第 9 章（績效評估）

## � 維護原則

1. **政策每年審查一次** — 配合 ISO 年度管理審查
2. **Service Catalog 半年更新一次** — 反映服務變更或新增
3. **L1-L5 成熟度每月評估** — 用於月度進度追蹤
4. **實施計劃每季檢討** — 調整優先級與資源配置

## ⚠️ 注意事項

### 不要做的事
❌ 不要為了稽核而創建文件 — 應先有服務，再有文件驗證  
❌ 不要跳過 L1 直接做 L4/L5 — 可靠性是底線  
❌ 不要忽視 L3（使用者體驗） — 人不用，所有投資都白費  
❌ 不要讓合規流程阻礙技術創新  
❌ 不要一次性追求所有層級達到滿分

### 應該做的事
✅ 優先採購成熟服務（L1-L3）— 自建投入核心差異化（L4）  
✅ 先穩定 L1（不中斷、不爆炸）  
✅ 建立 L2（出事有訊號、能處理）  
✅ 優化 L3（人用起來順）  
✅ 自動化 L4（減少浪費時間）  
✅ 最後整理 L5（文件與證書自然產生）
