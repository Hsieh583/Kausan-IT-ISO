# ISO 27001 資源與實施管理

## 概述

本目錄從**IT經理運營視角**管理ISO 27001實施所涉及的所有資源，包括人員、組織、軟體、設備、預算等。

與ISO文檔體系不同，本目錄聚焦於**資源配置、成本控制、進度管理、績效追蹤**等實務管理工作。

---

## 目錄結構

### 📋 01_人力資源
管理ISO 27001實施涉及的所有人員
- 團隊成員清冊與職務分工
- 技能與培訓需求
- 工時投入跟蹤
- 人員變動管理

### 🏢 02_組織結構
定義ISO 27001治理組織
- 組織圖與報告線
- 資訊安全委員會成員與職責
- 決策權限與層級
- 跨部門協調機制

### 💻 03_軟體與工具
記錄實施與運營所用的系統
- 工具清冊（Wazuh、Veeam、Nessus等）
- 授權與成本
- 部署狀態與升級計劃
- 技術支援合同

### 🖥️ 04_硬體與基礎設施
管理安全相關的硬體投資
- 伺服器與網路設備
- 存儲與備份設備
- 安全設備（防火牆、UAM等）
- 生命週期與淘汰計劃

### 💰 05_預算與成本
追蹤ISO 27001實施的財務投入
- 年度預算計劃
- 成本分類（軟體、硬體、人力、外包）
- 季度支出跟蹤
- ROI分析

### 📅 06_項目進度
監督實施進度與里程碑
- Phase 1-12 實施計劃
- 關鍵里程碑與截止日期
- 風險與延期管理
- 成果交付物

### 🤝 07_供應商管理
管理外部合作與服務商
- 軟體與硬體供應商清冊
- 維保合同與SLA
- 供應商績效評估
- 採購與續約管理

### 📊 08_績效指標
定義與追蹤關鍵績效指標
- 技術KPI（事件檢測率、響應時間等）
- 合規KPI（不符項改善率、審核通過率等）
- 運營KPI（可用性、備份成功率等）
- 月度與年度報告

### ⚠️ 09_風險與依賴
管理實施過程中的風險與瓶頸
- 關鍵依賴項（人、技術、資金）
- 風險識別與應對計劃
- 瓶頸與約束條件
- 應急預案

### 📚 10_文件治理與歷史
記錄文件體系的演進與治理決策
- 文件架構審查報告
- 文件整合與重組記錄
- 治理決策依據與歷史
- 文件體系演進過程

## 使用指南

### IT經理的工作流程

1. **規劃階段**
   - 參考06_項目進度中的Phase計劃
   - 在05_預算與成本中批准資源分配
   - 在07_供應商管理中簽訂合同

2. **執行階段**
   - 在01_人力資源中跟蹤團隊工時
   - 在03/04目錄中管理工具與硬體部署
   - 在06_項目進度中更新進度

3. **監控階段**
   - 在08_績效指標中檢視KPI達成情況
   - 在09_風險與依賴中評估風險
   - 定期召開資源協調會議

4. **優化階段**
   - 分析成本效益，優化預算分配
   - 評估供應商績效，調整合作
   - 識別技能缺口，規劃培訓

## 與ISO文檔體系的關係

| 視角 | 目錄 | 焦點 |
|------|------|------|
| **合規視角** | ISO27001_文檔體系 | 政策、程序、記錄 |
| **稽核視角** | 稽核與管理審查 | 檢查清單、報告 |
| **管理視角** | 資源與實施管理 | **人、組織、工具、預算** |

三個視角相互支撐，形成完整的ISO 27001管理系統。

## 關鍵決策點

### 資源投入決策
- 是否購買新工具（Wazuh Pro/企業版升級）
- 是否擴展團隊（招聘資安人員）
- 是否外包某些功能（滲透測試、稽核）

### 成本控制決策
- 軟體授權優化（批量購買、企業合同）
- 硬體升級時機（伺服器容量規劃）
- 內包vs外包的成本比較

### 進度調整決策
- 優先級重新排列（哪些Phase可加速）
- 資源重新分配（應對人員變動）
- 里程碑延期（應對技術或資金瓶頸）

| **每年** | 年度計劃、預算申請、策略調整 | IT經理、CEO、CFO |
