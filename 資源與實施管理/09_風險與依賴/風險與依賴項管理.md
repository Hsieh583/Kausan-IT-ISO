# ISO 27001 實施 - 風險與依賴項管理

## 概述

系統性地識別、評估、監控ISO 27001實施與運營過程中的風險，管理關鍵依賴項，確保項目順利推進。

---

## 風險管理框架

### 風險評估矩陣

```
影響度
5 (極高) │ █  █  █  █  █
         │ ───────────────
4 (高)   │ ▓  █  █  █  
         │ ───────────────
3 (中)   │ ░  ▓  █  █
         │ ───────────────
2 (低)   │    ░  ░  ▓
         │ ───────────────
1 (極低) │       ░  ░
         └───────────────────
           1  2  3  4  5
         (發生率)

█ = 極高風險 (立即處理)
▓ = 高風險 (密切監控)
░ = 中風險 (定期檢視)
  = 低風險 (正常監控)
```

## 核心風險清冊

### 1. 人力資源風險

#### 風險 HR-001: 關鍵人員離職 (CISO)

| 屬性 | 詳情 |
|------|------|
| **風險描述** | CISO是ISO實施核心領導者，若離職將導致項目領導真空、進度停滯 |
| **影響度** | ⭐⭐⭐⭐⭐ 極高 (項目無法推進) |
| **發生率** | ⭐⭐ 低 (~5%年度風險) |
| **風險值** | **中等** (高影響 x 低發生率) |
| **觸發指標** | • CISO提交辭呈 • 突然離職信號 |
| **預防措施** | • 制定留任激勵計劃 (2026年全年績效獎金15%) |
| | • 建立知識文檔體系，降低人員依賴 |
| | • 跨培訓資安合規專員，建立備用領導 |
| | • 建立繼任計劃 |
| **應急響應** | • 立即提升資安合規專員為代理CISO |
| | • 聘請外部顧問進行技術指導 |
| | • 可調整Phase時間表，優先完成關鍵工作 |
| **責任人** | IT經理 / HR主管 |
| **監控頻率** | 季度員工滿意度調查 |
| **狀態** | 🟢 控制良好 (目前無異常) |

| **風險值** | **中高** |
| **具體風險點** | 1. Okta ↔ Wazuh身份同步 (API相容性) |
| | 2. 防火牆日誌格式 → Wazuh解析 |
| | 3. Veeam ↔ Wazuh備份告警集成 |
| | 4. AD ↔ Okta同步延遲 |
| **預防措施** | ✅ 與各廠商簽署技術支援合同 (4小時響應) |
| | ✅ 提前進行POC驗證 (2月完成Okta-Wazuh) |
| | • 預留2周集成調試時間 |
| | • 詳細記錄集成步驟與問題 |
| **應急響應** | • 階段性集成 (優先集成關鍵路徑) |
| | • 保留備用方案 (Splunk / ELK替代) |
| | • 延期非關鍵集成至Phase 11 |
| **責任人** | 資安實施經理 / 網路安全工程師 |

| **狀態** | 🟡 進行中 (POC準備中) |

## 風險應對策略

### 四層應對策略

| 策略 | 說明 | 例子 |
|------|------|------|
| **避免** | 改變計劃以消除風險 | 選擇更穩定的廠商，避免不成熟技術 |
| **緩和** | 減少風險發生的可能性或影響 | 簽署SLA、購買保險、建立備用方案 |
| **轉移** | 將風險轉移給第三方 | 購買設備保修、外包服務合同 |
| **接受** | 承認風險並準備應對 | 建立應急基金、應急響應計劃 |

### 風險應對決策樹

```
識別風險
    │
    ├─ 影響度極高且發生率高?
    │   ├─ 是 → 避免或立即緩和
    │   └─ 否 → 下一步
    │
    ├─ 可以轉移? (保險、合同、外包)
    │   ├─ 是 → 評估成本效益
    │   └─ 否 → 下一步
    │
    ├─ 可以緩和?
    │   ├─ 是 → 實施緩和措施 + 定期監控
    │   └─ 否 → 接受風險
    │
    └─ 接受風險 + 建立應急計劃
```

## 應急響應計劃

### 關鍵事件應急流程

#### 情景1：CISO突然離職

```
T+0分鐘: 獲知信息
  → HR通知IT經理與CEO

T+30分鐘: 應急決策
  → 召集應急會議 (IT經理、CEO、HR)
  → 決定是否延期認證

T+2小時: 應急部署
  → 資安合規專員升任代理CISO
  → 聯絡備選外部顧問
  → 通知資安委員會

T+1天: 穩定與恢復
  → 明確代理CISO授權
  → 啟動CISO招聘 (急聘)
  → 調整Phase時間表

T+1週: 監控與調整
  → 代理CISO運行評估
  → 確認外部支援安排
  → 更新風險評估
```

#### 情景2：Wazuh服務中斷
