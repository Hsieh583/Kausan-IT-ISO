# ISO 27001 實施 - 組織結構與治理

## 概述

定義ISO 27001實施與運營的組織結構、決策機構、權責分工、跨部門協調機制。

---

## 組織圖

```
CEO / CFO
    │
    ├─ IT經理
    │   │
    │   ├─ CISO (資訊安全主管) ⭐
    │   │   ├─ 資安合規專員
    │   │   ├─ 資安實施經理
    │   │   └─ 資訊安全委員會秘書
    │   │
    │   ├─ 系統管理員
    │   │   ├─ Windows/Linux管理
    │   │   └─ 虛擬化/存儲管理
    │   │
    │   ├─ 網路安全工程師 ⭐
    │   │   ├─ SIEM (Wazuh)運營
    │   │   └─ 防火牆管理
    │   │
    │   └─ 備份與復原工程師
    │       └─ Veeam系統管理
    │
    ├─ HR主管
    │   └─ 員工背景調查 & 培訓協調
    │
    ├─ 財務主管
    │   └─ 預算批准 & 成本控制
    │
    ├─ 營運主管
    │   └─ 業務連續性 & 風險管理
    │
    └─ 法務主管
        └─ 合規 & 隱私 & 合同管理

⭐ = ISO 27001核心角色
```

## 關鍵決策機構

### 1. 資訊安全委員會 (Security Steering Committee)

#### 組成

| 職位 | 代表 | 部門 | 角色 |
|------|------|------|------|
| **主委** | CEO或COO | 最高管理層 | 主持會議、核准預算 |
| **副主委** | IT經理 | IT部門 | 執行管理、定期報告 |
| **委員** | CISO | IT部門 | 資安策略、政策制定 |
| **委員** | 財務主管 | 財務部 | 預算與成本 |
| **委員** | HR主管 | 人力資源 | 人員與培訓 |
| **委員** | 營運主管 | 營運部門 | 業務風險、BCP |
| **委員** | 法務主管 | 法務部 | 法規遵循、隱私 |
| **秘書** | 資安合規專員 | IT部門 | 會議記錄、行動追蹤 |

#### 職責

- 審核與批准資安政策與程序
- 批准年度資安預算與計劃
- 監督ISO 27001實施進度
- 評估資安風險與應對措施
- 審查內部稽核報告
- 進行管理評審 (年度)

#### 會議頻率

| 會議類型 | 頻率 | 議題 |
|---------|------|------|
| 定期會議 | 季度 (每季月初) | 進度、風險、決策 |
| 臨時會議 | 需要時 | 緊急決策 |
| 年度管理評審 | 1次/年 | 全面評估、戰略調整 |

**2026年會議日程**：
- Q1會議: 2026-02-XX (進度評估、Q1行動計劃)
- Q2會議: 2026-05-XX (Phase 10-11進度)
- Q3會議: 2026-08-XX (內部稽核結果)
- Q4會議: 2026-11-XX (年度評估、2027計劃)

## 職務描述

### IT經理

**職位代碼**：MGR-IT-001  
**報告給**：CEO / CFO  

#### 主要責任

1. **整體監督**
   - ISO 27001實施的總體責任
   - 資源分配與預算控制
   - 進度監督與風險管理
   - 向董事會報告

2. **組織管理**
   - 組建與管理資安團隊
   - 制定工作計劃
   - 績效評估與激勵

3. **決策支援**
   - 資安委員會秘書協調
   - 重大決策建議
   - 戰略規劃

#### 要求

- 5+年IT管理經驗
- 熟悉資訊安全管理體系
- 優秀的領導與溝通能力
- ISO 27001基礎知識

#### KPI

- ISO實施進度達成率 ≥ 95%
- 預算控制度 100% ± 5%
- 團隊滿意度 ≥ 4.0/5

---

### CISO (資訊安全主管)

**職位代碼**：CISO-001  
**報告給**：IT經理  

#### 主要責任

1. **戰略與政策**
   - 制定資訊安全政策
   - 風險評估與應對
   - 合規性確保

2. **技術指導**
   - 技術方案評審
   - 最佳實踐推廣
   - 架構決策

3. **管理與治理**
   - 資安委員會主導
   - 內部稽核組織
   - 管理評審參與

4. **外部代表**
   - 廠商協議談判
   - 認證審核協調
   - 監管部門溝通

#### 要求

- 10+年資訊安全經驗
- ISO 27001 Lead Auditor認證 (或計劃取得)
- 企業級安全系統經驗
- 卓越的領導能力

#### KPI

- 不符項改善率 100%
- 內部稽核不符項 ≤ 15項
- 政策簽訂率 ≥ 95%
- 員工培訓滿意度 ≥ 4.2/5

---

## 溝通與上報流程

### 信息流向

```
技術問題
    ↓
資安實施經理 (每日跟進)
    ↓
CISO (每週評估)
    ↓
IT經理 (月度報告)
    ↓
資安委員會 (季度報告)
    ↓
CEO / CFO (重大事項)
```

### 報告內容與頻率

| 報告 | 發送人 | 接收人 | 頻率 | 內容 |
|------|--------|--------|------|------|
| 進度報告 | 資安實施經理 | CISO + IT經理 | 每週 | Phase進度、風險、障礙 |
| 月度總結 | CISO | 資安委員會 | 月度 | KPI、問題、決策項 |
| 季度評估 | IT經理 | 資安委員會 | 季度 | 整體進度、成本、風險 |
| 年度總結 | CISO | 董事會 | 年度 | 成果、建議、2027計劃 |

## 跨部門責任矩陣 (RACI)

| 工作項 | CISO | 資安合規 | 資安實施 | IT經理 | HR | 財務 | 營運 | 法務 |
|--------|------|---------|---------|--------|-----|------|------|------|
| 政策制定 | R/A | C | C | I | C | I | C | C |
| 記錄模板 | I | R/A | C | I | - | - | - | I |
| 培訓組織 | I | A | R | I | R | I | C | I |
| 預算批准 | C | - | - | R | - | A | - | - |
| 事件響應 | R | C | R | I | - | - | R | C |
| 內部稽核 | R | R/A | C | I | C | - | C | C |
| 合規報告 | R/A | R | C | C | - | I | I | C |
| 技術實施 | I | C | R/A | I | - | - | - | - |

**說明**：
- **R (Responsible)**: 執行者
- **A (Accountable)**: 問責人 (最終決策者)
- **C (Consulted)**: 諮詢人 (提供意見)
- **I (Informed)**: 通知人 (被告知結果)

## 管理審查流程

### 年度管理審查

**時間**：每年12月  
**主持**：CISO  
**與會**：資安委員會全體成員

**審查內容**：
1. 政策與目標的實現情況
2. 資安事件與改善情況
3. 利益相關方的意見
4. 風險與機遇評估
5. 資源與能力評估
6. 下年度計劃確認

**産出**：
- 管理審查報告
- 改善計劃
- 新年度目標與政策調整
