# IT 活動與專案計畫 (2026-2028)

> **組織**：高倉物流股份有限公司  
> **計畫期間**：2026年1月 - 2028年12月  
> **目的**：提升IT服務品質與業務支援能力

---

## 2026年 - IT基礎強化年

### Q1 (2026年1-3月)

#### 2026.01 - IT人力招募專案
- **活動**：資安合規專員招募與到職
- **為什麼**：現有IT經理工作負荷205%，急需專人處理日常資安監控與合規工作
- **預期成果**：1名合規專員到位，分擔每日Wazuh監控、帳號管理、事件記錄等工作

#### 2026.01-02 - DR異地備份專案啟動
- **活動**：災難復原計畫建置，異地備份驗證
- **為什麼**：確保業務資料安全，萬一主機房故障能快速復原
- **預期成果**：建立異地備份機制，復原時間目標(RTO) < 4小時

#### 2026.02-04 - Wazuh SIEM系統強化
- **活動**：Wazuh安全監控平台優化、整合所有日誌來源
- **為什麼**：統一監控所有系統（AD、防火牆、VPN、NAS、CCTV），早期發現異常
- **預期成果**：
  - 整合15+個日誌來源
  - 建立50+條告警規則
  - 減少誤報率至5%以下
  - 資安合規專員每日只需查看Wazuh儀表板

#### 2026.02-03 - 內部稽核制度建立
- **活動**：完成第一次完整內部稽核（存取控制、密碼管理、政策執行）
- **為什麼**：驗證現有制度是否有效執行，發現改善點
- **預期成果**：產出稽核報告，識別5-10項改善事項

### Q2 (2026年4-6月)

#### 2026.04-06 - HA高可用性系統建置
- **活動**：關鍵業務系統導入高可用性架構（Active-Standby或Load Balancing）
- **為什麼**：減少系統停機時間，提升服務連續性
- **目標系統**：
  - AD網域控制器（雙DC）
  - 核心應用伺服器
  - 資料庫SQL Server
- **預期成果**：關鍵系統可用性 > 99.5%

#### 2026.05-06 - 帳號生命週期自動化
- **活動**：優化Okta IAM系統，實現帳號自動化管理
- **為什麼**：減少人工作業錯誤，加速新人到職/離職流程
- **預期成果**：
  - 新人到職當天完成所有帳號開通
  - 離職當天自動停用所有權限
  - 減少IT人工作業50%

#### 2026.06 - 弱點掃描與修補流程優化
- **活動**：建立自動化弱點掃描排程，修補追蹤機制
- **為什麼**：及早發現系統漏洞，降低被攻擊風險
- **預期成果**：
  - 每月自動掃描所有伺服器
  - 高風險漏洞7天內修補
  - 中風險漏洞30天內修補

## 總結

這份計畫以**提升IT服務品質**為核心，透過系統性的專案推動，逐步建立：

1. **穩定可靠的基礎設施**（HA、備份、監控）
2. **主動式的資安防禦**（Wazuh、弱點管理、員工培訓）
3. **高效率的IT服務**（自動化、工單系統、雲端服務）
4. **持續改善的文化**（內部稽核、KPI追蹤、外部驗證）

ISO 27001認證是這些努力的自然成果，而非最終目標。我們的目標是**讓IT成為業務成長的助力，而非阻力**。
